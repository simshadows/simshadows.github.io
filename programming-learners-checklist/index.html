C Language Fundamentals

Chances are, the language you started with has made things very convenient for you. I think that every serious programmer should eventually try learning how to use C because C doesn't hold your hand (nearly as much). C will give you so much appreciation for all the bells and whistles other languages give you and give you a more critical understanding of algorithm and data structure performance.

On top of that, C is a great stepping stone into learning C++, gives you great background for assembly language and digital circuit design (trust me, it will make sense when you get there), and practically necessary to learn about Operating Systems, other low-level/systems programming work, and deep security topics.

I very much highly recommend learning C as your first language if you can, though it's understandably not for everyone because it's a very tedious language to work with. If you don't like it, there's no harm in holding it off for later when you're more motivated.

  • COREQUISITES: Programming Fundamentals; Statically Typed Programming Fundamentals (Corequisite means you need to learn these as well, but they can be learned together.)

[OPTIONAL] C++

C++ deserves its own section because it's its own beast that's very different to most popular imperative languages. While you can get away with neglecting a lot of things, ignoring the subtleties will eventually be very painful. It is very easy to shoot yourself in the foot here.

While I don't consider C to be a strict prerequisite, I very highly recommend learning C and doing a wide range of data structures and algorithms exercises in C before starting C++. Doing all sorts of HackerRank exercises and such is amazing for drilling in the most critical concepts in C++. But if you don't want to go the C route, you can still learn everything you need starting with C++ (and that knowledge should actually carry to C if you ever need to do C).

  • PREREQUISITES: Data Structures and Algorithms
  • COREQUISITES: Statically Typed Programming Fundamentals; Object Oriented Programming
  • RECOMMENDED PREREQUISITES: C Language Fundamentals
  • Know everything from the C Language Fundamentals section, within the context of C++. (C++ is mostly a massive superset of C, and everything about C is extremely useful background for C++.)
  • Know everything from the Statically Typed Programming Fundamentals section, within the context of C++. (C++ is a statically typed language with a very complicated type system.)
  • Know everything from the Object Oriented Programming section, within the context of C++. (C++ is also an object oriented language, and it's quite important here.)
  • Know pointers and references, and the differences between them.
  • Know lvalue/rvalue, copy/move semantics, specifically copy/move constructors. Know how all these concepts relate to each other and the differences between them.
